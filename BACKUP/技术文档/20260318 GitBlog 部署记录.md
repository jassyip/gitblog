## 📖 一、项目背景
一直想搞一个自己的博客，但又不想折腾服务器、数据库、域名这些东西——太繁琐了，维护成本也高。

- 📝 用 **GitHub Issues** 写文章
- ⚙️ 用 **GitHub Actions** 自动构建
- 🌐 用 **GitHub Pages** 免费托管
- 🐍 用 **Python** 生成静态页面
整套流程完全跑在 GitHub 生态里，零服务器成本，写文章就像提 Issue 一样简单。对于我这种不想运维、只想写写东西的人来说，简直是完美方案。

## 🛠 二、部署步骤
### 1. Fork 仓库
前往 [yihong0618/gitblog](https://github.com/yihong0618/gitblog)，点击右上角 **Fork**，将仓库 Fork 到自己的 GitHub 账号下。
Fork 完成后，仓库地址变为：
```javascript
https://github.com//gitblog
```
> 💡 **注意：** Fork 时建议保持仓库名为 `gitblog`，后续配置更顺畅。
---
### 2. 配置 GitHub Secrets（G_T Token）
GitHub Actions 需要权限来自动提交生成的静态文件，因此需要配置一个 **Personal Access Token（PAT）**。
**步骤如下：**
1. 前往 GitHub → **Settings → Developer settings → Personal access tokens → Tokens (classic)**
2. 生成一个新 Token，勾选 `repo` 权限
3. 复制生成的 Token（只显示一次，记得保存！）
4. 回到你 Fork 的 `gitblog` 仓库
5. 进入 **Settings → Secrets and variables → Actions**
6. 点击 **New repository secret**，填写：
	- **Name：** `G_T`
	- **Value：** 粘贴刚才复制的 Token
✅ 保存完成，Actions 就有了推送权限。
---
### 3. 修改个人配置文件
需要编辑两个文件，填入自己的信息：
#### `main.py`
找到文件顶部的配置区域，修改以下内容：
```python
GITHUB_NAME = "你的 GitHub 用户名"
GITHUB_EMAIL = "你的邮箱"
REPO_NAME = "你的用户名/gitblog"
```
#### `config.toml`
```toml
[blog]
title = "你的博客标题"
subtitle = "副标题"
footer = "Powered by GitBlog"
```
> 💡 这两个文件直接在 GitHub 网页端编辑提交即可，无需本地克隆。
---
### 4. 启用 GitHub Pages
1. 进入仓库 **Settings → Pages**
2. **Source** 选择 **GitHub Actions**
3. 保存
首次启用后，Actions 会自动触发一次构建流程。稍等几分钟，博客就会上线在：
```javascript
https://.github.io/gitblog/
```

## ✅ 三、部署成果
经过以上步骤，博客已成功上线！🎉
