## 二、核心功能
### 2.1 语音与文本日程创建
用户可通过语音或文字直接描述日程安排，系统自动解析事件标题、时间、地点、参与者及备注等要素，并在对应平台创建日历事件或任务条目。对于语义模糊或信息不完整的输入，系统将主动询问补充，确保事件信息准确无误后再行创建。
### 2.2 智能提醒调度
系统根据事件类型与紧急程度，自动计算并设置最合适的提前提醒时间。例如：普通会议默认提前 15 分钟提醒，重要会议提前 30 分钟，航班及长途出行提前 3 小时，医疗预约提前 1 小时。提醒规则可由用户自定义，系统将记忆偏好并在后续事件中自动应用。
### 2.3 每日早间日程预览
每天早上 08:00，系统主动推送当日日程概览，内容包括全部事件列表、时间安排、重要注意事项（如需提前出发、需携带材料等）以及与昨日相比的变动提示。推送时间可由用户根据个人作息调整。
### 2.4 冲突检测
每次创建或修改日程时，系统自动检查新事件与已有安排之间是否存在时间冲突。若检测到冲突，系统将明确说明冲突详情，并提供可行的调整建议（如推迟时间、缩短时长或重新安排），由用户确认后再执行操作。
### 2.5 多平台智能同步
系统支持与以下六个平台进行智能同步，用户按需连接，未连接的平台自动跳过，不影响其他功能正常运行：
1. Google Calendar：将事件同步至 Google 日历，保持跨设备可见性。
2. Outlook Calendar：将事件同步至 Outlook 日历，适配企业及 Microsoft 生态用户。
3. Notion：在用户指定的 Notion 数据库中维护结构化日程日志，支持长期归档与检索。
4. Todoist：将具有明确截止时间的事件同步创建为 Todoist 任务，便于跨平台任务管理。
5. Asana：将相关事件同步创建为 Asana 任务，适配团队协作场景。
6. Linear：将技术类事件或项目节点同步创建为 Linear Issue，适配研发团队工作流。
### 2.6 多语言自适应
系统支持全球化语言自适应（Match the user's primary language）。所有提醒文本、早间预览、冲突提示及引导消息，均自动识别用户的母语偏好，以对应语言输出，无需手动配置。

## 五、英文 Context（配方完整描述）
PlanSync is an AI-powered schedule management assistant designed to unify event creation, intelligent reminders, conflict detection, and multi-platform synchronization into a seamless daily workflow. The recipe supports both voice and text input and integrates with up to six platforms based on the user's connected services.
Key capabilities include:
1. Voice and text schedule creation: Users describe events naturally through voice messages or text. The system parses key elements including event title, date and time, location, participants, and notes. If any required information is ambiguous or missing, the system proactively requests clarification before creating the event.
2. Intelligent reminder scheduling: Reminders are automatically scheduled based on event type and urgency. Default lead times include 15 minutes for standard meetings, 30 minutes for important meetings, 3 hours for flights and long-distance travel, and 1 hour for medical appointments. Reminder preferences are remembered and applied automatically to future events of the same type.
3. Daily morning schedule preview at 08:00: A structured daily overview is delivered each morning at 8:00 AM by default, covering all events for the day, key notes and preparation reminders, and any changes from the previous day. The delivery time is configurable.
4. Conflict detection: When creating or modifying an event, the system automatically checks for scheduling conflicts with existing entries. If a conflict is detected, the system clearly explains the overlap and offers actionable adjustment suggestions for the user to confirm before proceeding.
5. Multi-platform intelligent sync: The system supports synchronization with six platforms. Each platform is activated only if the user has connected it; unconnected platforms are silently skipped without affecting other functionality.
	- If Google Calendar is connected, sync events to Google Calendar.
	- If Outlook Calendar is connected, sync events to Outlook Calendar.
	- If Notion is connected, maintain a structured schedule log in the user's designated Notion database.
	- If Todoist is connected, create corresponding tasks in Todoist for time-bound events.
	- If Asana is connected, create tasks in Asana for relevant events.
	- If Linear is connected, create issues in Linear for technical milestones or project-related events.
	- Skip any platform that is not connected.
6. Adaptive communication style (Match the user's chat habits): Poke analyzes the user's historical interaction patterns to identify their preferred communication style, including language, tone, level of formality, and verbosity. All reminders, previews, and proactive messages are dynamically adjusted to match these habits, ensuring a natural and personally resonant experience.
7. Multilingual adaptive output (Match the user's primary language): All messages, reminders, conflict notifications, and guided content are delivered in the user's primary language. Language detection is based on interaction history and system language settings. Supported languages include but are not limited to Simplified Chinese, Traditional Chinese, English, and Japanese.
## 六、设计原则
1. 低摩擦创建：以自然语言输入为核心，最大程度减少用户手动填写表单的操作负担。
2. 按需集成：多平台同步以可选形式存在，用户连接哪个平台即激活对应模块，未连接时系统正常运行。
3. 主动服务：每日早间预览与智能提醒均由系统主动发起，而非依赖用户手动查询。
4. 冲突透明化：所有日程冲突均明确呈现并提供建议，最终决策权始终归属用户。
5. 全局化适配：语言、时区与通知格式均以用户实际环境为准，无需额外配置。
